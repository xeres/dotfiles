# ADR-0006 zsh プラグイン管理に zinit を使用する

## 🤔 背景

ADR-0005 で zsh をログインシェルとして採用した。zsh プラグインを効率的に管理するためのプラグインマネージャを選定する必要がある。プラグイン数が増えてもシェルの起動速度を維持することが重要な要件である。

## 💡 決定事項

zsh のプラグイン管理に zinit を使用する。

### 決定に至った差別化要素

zinit は遅延ロード（turbo mode）をネイティブにサポートしており、シェル起動速度を犠牲にせずに多数のプラグインを利用できる。oh-my-zsh や sheldon と比較して、プラグインのロードタイミングを細粒度で制御できるためパフォーマンスに優れる。

## 🔍 検討した選択肢

|  | 起動速度 | プラグイン数 | 遅延ロード | 設定の柔軟性 |
| --- | --- | --- | --- | --- |
| zinit | ✅ turbo mode | ✅ 任意の GitHub リポジトリ | ✅ ネイティブ対応 | ✅ 細粒度制御 |
| oh-my-zsh | ❌ 全プラグイン即時ロード | ✅ 豊富な同梱プラグイン | ❌ 非対応 | ⚠️ フレームワーク依存 |
| sheldon | ✅ Rust 実装で高速 | ✅ 任意の GitHub リポジトリ | ⚠️ defer で部分対応 | ⚠️ TOML 設定 |
| zplug | ⚠️ やや遅い | ✅ 任意の GitHub リポジトリ | ⚠️ defer で部分対応 | ✅ 柔軟 |

### 1️⃣ oh-my-zsh

最も普及している zsh フレームワーク。テーマやプラグインを多数同梱する。

- ❌ 起動速度: 有効化したプラグインをすべて即時ロードするため、プラグインが増えると起動が遅くなる
- ✅ プラグイン数: 同梱プラグインが豊富で設定なしに利用できる
- ❌ 遅延ロード: ネイティブの遅延ロード機構がない
- ⚠️ 設定の柔軟性: フレームワークの規約に従う必要がある

### 2️⃣ sheldon

Rust 実装のプラグインマネージャ。TOML ファイルで宣言的にプラグインを管理する。

- ✅ 起動速度: Rust 実装でプラグイン解決自体は高速
- ✅ プラグイン数: 任意の GitHub リポジトリからインストール可能
- ⚠️ 遅延ロード: defer で部分的に対応するが、zinit の turbo mode ほど細粒度ではない
- ⚠️ 設定の柔軟性: TOML 設定で宣言的だが、ロードタイミングの細かい制御は限定的

### 3️⃣ zplug

zsh プラグインマネージャの先駆け的存在。zinit と類似した構文を持つ。

- ⚠️ 起動速度: プラグイン解決がやや遅い
- ✅ プラグイン数: 任意の GitHub リポジトリからインストール可能
- ⚠️ 遅延ロード: defer オプションで部分的に対応
- ✅ 設定の柔軟性: 柔軟なロードオプションを持つ
