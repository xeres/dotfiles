# ADR-0001 chezmoi を dotfiles 管理ツールとして採用する

## 🤔 背景

複数の環境（個人マシン、開発サーバー、CI 等）にわたって一貫したシェル設定・ツール設定を維持するには、dotfiles の管理ツールが必要になる。環境ごとに OS やインストール済みパッケージが異なるため、単純なファイルコピーではなく環境差分を吸収する仕組みが求められる。

## 💡 決定事項

dotfiles の管理ツールとして chezmoi を採用する。

### 決定に至った差別化要素

chezmoi は Go Template をネイティブにサポートしており、OS・ホスト名・ユーザー名などの環境変数に応じてファイル内容を動的に切り替えられる。これにより、単一のソースリポジトリから複数環境への展開が可能になり、環境差分をテンプレート内で宣言的に管理できる。

他の選択肢では環境差分の吸収にシェルスクリプトや手動の分岐が必要になるのに対し、chezmoi はこれをツール自体の機能として提供しているため、設定の可読性と保守性が高い。

## 🔍 検討した選択肢

|  | テンプレートエンジン | 環境差分吸収 | 学習コスト | 依存の少なさ |
| --- | --- | --- | --- | --- |
| chezmoi | ✅ Go Template 内蔵 | ✅ ネイティブ対応 | ⚠️ 独自概念あり | ✅ シングルバイナリ |
| GNU Stow | ❌ なし | ❌ シンボリックリンクのみ | ✅ 単純 | ✅ Perl のみ |
| yadm | ⚠️ Jinja2（alt files） | ⚠️ alt files + テンプレート | ✅ git 知識で足りる | ⚠️ git + Python |
| 手動スクリプト | ⚠️ 自作が必要 | ⚠️ スクリプト次第 | ✅ 自由度高い | ❌ 保守コスト大 |

### 1️⃣ GNU Stow

シンボリックリンクを作成する Unix 伝統のアプローチ。設定ファイルをパッケージ単位でディレクトリに分けてリンクを張る。

- ❌ テンプレートエンジン: なし。環境差分には対応できない
- ❌ 環境差分吸収: 全環境で同一ファイルを使うか、手動で分岐する必要がある
- ✅ 学習コスト: 概念が単純で覚えることが少ない
- ✅ 依存の少なさ: Perl のみで動作する

### 2️⃣ yadm

git リポジトリをそのままホームディレクトリに重ねる git ラッパー型ツール。Jinja2 テンプレートや alt files による環境分岐機能を持つ。

- ⚠️ テンプレートエンジン: Jinja2 を利用可能だが、Python 依存が発生する
- ⚠️ 環境差分吸収: alt files でファイル単位の切り替えは可能だが、ファイル内の部分的な差分管理は限定的
- ✅ 学習コスト: git の知識があれば基本操作は容易
- ⚠️ 依存の少なさ: git + テンプレート利用時は Python が必要

### 3️⃣ 手動スクリプト（install.sh 等）

シェルスクリプトでファイルのコピーやリンク作成を自前で実装する方法。

- ⚠️ テンプレートエンジン: sed/envsubst 等で自作は可能だが標準化されていない
- ⚠️ 環境差分吸収: 自由度は高いがスクリプトの肥大化を招く
- ✅ 学習コスト: 外部ツールの学習は不要
- ❌ 依存の少なさ: スクリプト自体の保守コストが継続的に発生する
