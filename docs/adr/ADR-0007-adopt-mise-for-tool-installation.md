# ADR-0007 ツールのインストールに原則 mise を使用する

## 🤔 背景

開発に必要なツール（言語ランタイム、CLI ツール等）のバージョン管理とインストール方法を統一したい。環境ごとに異なるパッケージマネージャ（brew, apt 等）を使うとインストール手順が分散し、再現性が下がる。プロジェクト単位でツールのバージョンを固定し、チームメンバーや CI 環境で同一のバージョンを使える仕組みが必要になる。

## 💡 決定事項

ツールのインストールに原則として mise を使用する。

### 決定に至った差別化要素

mise は asdf 互換のプラグインエコシステムをそのまま利用でき、既存の asdf プラグイン資産を活かせる。加えてシングルバイナリで配布されるため、mise 自体のインストールに追加の依存（bash、プラグインスクリプト群）が不要で、chezmoi の bootstrap スクリプトから容易にセットアップできる。asdf と比べて UX が改善されており、`.tool-versions` と `mise.toml` の両方をサポートする柔軟性もある。

## 🔍 検討した選択肢

|  | asdf 互換 | 依存の少なさ | UX | 設定形式 |
| --- | --- | --- | --- | --- |
| mise | ✅ プラグイン互換 | ✅ シングルバイナリ | ✅ 改善された CLI | ✅ TOML + .tool-versions |
| asdf | ✅ 本家 | ❌ bash + プラグインスクリプト | ⚠️ やや冗長 | ⚠️ .tool-versions のみ |

### 1️⃣ asdf

シェルスクリプトベースの多言語バージョンマネージャ。プラグインシステムで拡張する。

- ✅ asdf 互換: 本家そのもの
- ❌ 依存の少なさ: bash、curl、git 等が必要。プラグインごとに追加の依存がある場合もある
- ⚠️ UX: コマンド体系がやや冗長で、エラーメッセージが分かりにくいことがある
- ⚠️ 設定形式: `.tool-versions` のみ対応
