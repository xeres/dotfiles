# ADR-0004 bats による非対話的テスト(CI)を採用する

## 🤔 背景

dotfiles の適用結果を対話的に確認する方法（ADR-0003）に加え、CI で自動実行できる非対話的なテストが必要になる。chezmoi apply 後のファイル配置やシェル設定が期待通りかを機械的に検証することで、変更のリグレッションを防ぎたい。

## 💡 決定事項

非対話的テストフレームワークとして bats（Bash Automated Testing System）を採用する。chezmoi apply 後の環境に対してシェルコマンドベースのアサーションを実行し、CI パイプラインで自動検証する。

### 決定に至った差別化要素

bats はシェルスクリプトのテストに特化しており、dotfiles のテスト対象（ファイルの存在確認、シェル変数、コマンド出力）と自然に適合する。テストの記述が bash そのものであるため追加の言語ランタイムが不要で、Docker コンテナ内（ADR-0003）でも apt で即座にインストールできる。

## 🔍 検討した選択肢

|  | シェルとの親和性 | 追加依存 | CI 統合 | エコシステム |
| --- | --- | --- | --- | --- |
| bats | ✅ bash ネイティブ | ✅ なし | ✅ 終了コードで判定 | ⚠️ 小規模 |
| shunit2 | ✅ POSIX sh 対応 | ✅ なし | ✅ 終了コードで判定 | ⚠️ 小規模 |
| pytest + subprocess | ⚠️ Python 経由 | ❌ Python 必要 | ✅ 豊富なプラグイン | ✅ 大規模 |

### 1️⃣ shunit2

xUnit スタイルの POSIX sh テストフレームワーク。

- ✅ シェルとの親和性: POSIX sh 互換で幅広いシェルで動作する
- ✅ 追加依存: シェルスクリプトのみで動作
- ✅ CI 統合: 終了コードベースで CI と統合可能
- ⚠️ エコシステム: bats と比べてテストヘルパーライブラリが少ない

### 2️⃣ pytest + subprocess

Python のテストフレームワークからシェルコマンドを subprocess で実行して検証する方法。

- ⚠️ シェルとの親和性: Python を経由するため間接的
- ❌ 追加依存: Python ランタイムと pytest のインストールが必要
- ✅ CI 統合: pytest の豊富なプラグインエコシステムが利用可能
- ✅ エコシステム: 大規模で成熟している
