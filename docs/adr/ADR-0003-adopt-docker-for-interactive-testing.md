# ADR-0003 Docker コンテナで対話的テスト環境を構築する

## 🤔 背景

dotfiles の変更をホスト環境に直接適用してテストすると、既存の設定を破壊するリスクがある。安全にテストするには、クリーンな状態から `chezmoi apply` を実行し、その結果を対話的に確認できる隔離環境が必要になる。

## 💡 決定事項

Docker コンテナを対話的テスト環境として使用する。`Dockerfile` でクリーンな Ubuntu 環境に chezmoi をインストール・適用し、`docker run --interactive --tty` で対話的にシェルに入って動作確認を行う。`Makefile` でビルド・実行を `make docker` に集約する。

### 決定に至った差別化要素

Docker は環境の再現性が高く、ホスト環境を汚さずに何度でもクリーンな状態からテストできる。ソースディレクトリをボリュームマウントすることで、ホスト側の変更を即座にコンテナ内に反映してイテレーションを回せる。VM と比べて起動が高速で、日常的な動作確認に適している。

## 🔍 検討した選択肢

|  | 隔離性 | 起動速度 | 再現性 | セットアップコスト |
| --- | --- | --- | --- | --- |
| Docker コンテナ | ✅ 完全隔離 | ✅ 秒単位 | ✅ Dockerfile で定義 | ⚠️ Dockerfile 作成 |
| VM（Vagrant 等） | ✅ 完全隔離 | ❌ 分単位 | ✅ Vagrantfile で定義 | ❌ VM 環境が必要 |
| ホスト環境で直接テスト | ❌ 隔離なし | ✅ 即時 | ❌ 状態依存 | ✅ 不要 |

### 1️⃣ VM（Vagrant 等）

仮想マシンを使って完全に隔離された OS 環境でテストする方法。

- ✅ 隔離性: カーネルレベルから隔離される
- ❌ 起動速度: VM の起動に数十秒〜数分かかり、試行錯誤のサイクルが遅い
- ✅ 再現性: Vagrantfile で環境を定義可能
- ❌ セットアップコスト: VirtualBox 等のハイパーバイザーが必要

### 2️⃣ ホスト環境で直接テスト

開発マシン上で直接 `chezmoi apply` を実行してテストする方法。

- ❌ 隔離性: 既存の設定を上書きするリスクがある
- ✅ 起動速度: 追加の起動時間なし
- ❌ 再現性: テスト前の状態に依存し、結果が安定しない
- ✅ セットアップコスト: 追加セットアップ不要
