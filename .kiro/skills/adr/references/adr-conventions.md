# ADR記述規約

ADRを作成する際の各セクションの書き方、ステータス管理、ファイル命名の規約。

## ファイル命名規約

- デフォルト形式: `ADR-NNNN-kebab-case-title.md`
  - `NNNN` は4桁ゼロ埋めの連番
  - タイトル部分はkebab-caseで、決定内容を端的に表す
  - 例: `ADR-0001-use-dynamodb-for-primary-db.md`
- 既存ADRがある場合はそのフォーマットに合わせる
  - プレフィクスの有無、ゼロ埋め桁数、日付形式の使用などは既存に揃える

## ステータス

| ステータス | 意味 |
| --- | --- |
| `Proposed` | 提案中。まだ合意に至っていない |
| `Accepted` | 採用決定。実装・運用に反映する |
| `Deprecated` | 廃止。置き換え先のADRはない |
| `Superseded by ADR-MMMM` | 別のADRで置き換えられた |

### ステータス遷移

- `Proposed` → `Accepted`: レビューを経て合意に至った
- `Accepted` → `Deprecated`: 不要になったが、代替の決定はない
- `Accepted` → `Superseded by ADR-MMMM`: 新しいADRで置き換えられた

新規作成時の初期ステータスは原則 `Proposed`。ユーザーが「決定済み」と明言した場合、または既存ADRをsupersedeする形で合意が取れている場合のみ `Accepted` とする。

## 各セクションの書き方

### 🤔 背景

- この決定が必要になった背景、解決しようとしている課題、前提条件、制約
- 「なぜ今この決定をするのか」を読者に伝える
- 決定そのものや選択肢はここに書かない（「決定事項」「検討した選択肢」に書く）
- 時系列ではなく論点ごとに整理する

### 💡 決定事項

- **能動態・現在形**で記述する
  - OK: 「プライマリDBとしてDynamoDBを採用する」
  - NG: 「DynamoDBを採用することに決めた」「採用します」
- 一つのADRには**一つの決定**のみを書く。複数の独立した決定は別のADRに分ける

#### 決定に至った差別化要素

- 他の選択肢と比較したうえで、採用案を選ぶ決め手となった要素を記述する
- **メリットは必ず**記入する
- **デメリットは存在する場合のみ**記入する（トレードオフを隠さない）
- 箇条書きではなく**自然言語の文章体**を中心に記述する
- 将来の読者が「なぜこの決定を変えたくなるか」を理解できるように書く

### 🔍 検討した選択肢

- 実際に検討した選択肢を書く（採用案を引き立てるためだけの、実際には検討していない選択肢は書かない）
- セクション冒頭に**縦軸に選択肢、横軸に検討軸のtable**を記載する
  - 各セルは3段階評価の絵文字（✅ / ⚠️ / ❌）で記入
  - セルに簡単な補足があれば併記する
  - **選択肢の数が検討軸の数より大幅に多い場合**（目安: 選択肢数 ≧ 検討軸数 × 2）は、横長になって読みにくくなるため、縦横を入れ替えてもよい（縦軸=検討軸、横軸=選択肢）
- tableの下に選択肢ごとに小見出し（`### 1️⃣ ...`, `### 2️⃣ ...`）を設け、詳細を記述する
  - 選択肢の概要は自然言語の文章体で書く
  - tableに記載した検討軸の評価を絵文字付きのリスト形式で再掲する
- 検討した選択肢がない（採用案以外に合理的な代替がなかった）場合はセクションごと削除してよい

### 🔗 関連ADR

- `Supersedes`: このADRが置き換える既存ADR
- `Superseded by`: このADRを置き換えた新しいADR（後から追記される）
- `Related`: 文脈的に関連するADR（supersede関係ではないもの）

### 📄 参考資料

- 参照した外部ドキュメント、設計資料、Issue/PRリンクなどを列挙する
- 検討の根拠となった一次情報へのリンクを優先する

## 既存ADRのsupersede手順

新規ADR `ADR-MMMM` が既存ADR `ADR-NNNN` を置き換える場合、両方を更新する:

**新規ADR (`ADR-MMMM`)**:

```markdown
---
status: Accepted
---

## 🔗 関連ADR

- Supersedes: ADR-NNNN <既存ADRのタイトル>
```

**既存ADR (`ADR-NNNN`)**:

```markdown
---
status: Superseded by ADR-MMMM
---

## 🔗 関連ADR

- Superseded by: ADR-MMMM <新規ADRのタイトル>
```

既存ADRの本文（背景・決定事項・検討した選択肢など）は**書き換えない**。歴史的記録として残す。
